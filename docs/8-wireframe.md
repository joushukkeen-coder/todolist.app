## 1. 개요

### 목적

TodoListApp v1 MVP의 모든 화면에 대한 레이아웃 구조, 인터랙션 흐름, 입력 제약 및 화면 전이를 ASCII 와이어프레임으로 문서화한다.

### 설계 기준

| 항목            | 기준                                                    |
|-----------------|---------------------------------------------------------|
| 데스크톱 기준   | 뷰포트 너비 ≥ 1024px, 사이드바 + 메인 콘텐츠 2단 레이아웃 |
| 모바일 기준     | 뷰포트 너비 < 1024px, 단일 컬럼 + 하단 네비게이션       |
| 인증 전 접근    | 로그인·회원가입 화면으로 리다이렉트                     |
| 토큰 저장       | Zustand authStore 메모리 저장, 새로고침 시 재로그인 필요 |

### 표기 규약

| 표기            | 의미             |
|-----------------|------------------|
| `[___________]` | 텍스트 입력 필드 |
| `[ 버튼명 ]`    | 클릭 가능한 버튼 |
| `[ ]`           | 체크박스 (미선택) |
| `[x]`           | 체크박스 (선택됨) |
| `[ 옵션 ▼ ]`   | 드롭다운 셀렉트  |
| `(●)`           | 라디오 버튼 (선택됨) |
| `( )`           | 라디오 버튼 (미선택) |
| `████`          | 색상 스와치      |
| `※`             | 안내/제약 사항   |
| `!`             | 에러 메시지      |
| `░░░░`          | 스켈레톤 로딩 블록 |
| `⟳`             | 스피너 (로딩 중) |

## 2. 화면 목록 및 사이트맵

```
TodoListApp
├── [비인증 영역]
│   ├── SCR-01  회원가입 화면
│   └── SCR-02  로그인 화면
│
└── [인증 영역] (JWT 필수)
    ├── SCR-03  할일 목록 화면 (메인 대시보드)
    │   └── SCR-04  할일 등록 모달
    │   └── SCR-05  할일 수정 모달
    │   └── SCR-06  할일 삭제 확인 모달
    ├── SCR-07  카테고리 관리 화면
    │   └── SCR-08  카테고리 생성/수정 모달
    │   └── SCR-09  카테고리 삭제 확인 모달
    └── SCR-10  프로필 설정 화면
        └── SCR-11  회원 탈퇴 확인 모달
```

## 3. 공통 컴포넌트

### 3.1 헤더 (데스크톱 ≥ 1024px)

```
+------------------------------------------------------------------------------+
|  TodoListApp                                        [사용자명]  [ 로그아웃 ] |
+------------------------------------------------------------------------------+
```

- 좌측: 앱 로고 및 명칭 (클릭 시 할일 목록으로 이동)
- 우측: 현재 로그인 사용자 이름 표시, 로그아웃 버튼
- 로그아웃 클릭 시 Zustand authStore 초기화 후 로그인 화면 이동

### 3.2 사이드바 (데스크톱 ≥ 1024px)

```
+--------------------+
|  내비게이션        |
+--------------------+
|  [ ] 할일 목록     |  <- 활성 시 강조 표시
|  [ ] 카테고리 관리 |
|  [ ] 프로필 설정   |
+--------------------+
```

- 너비: 약 200px 고정
- 현재 활성 화면의 메뉴 항목은 시각적으로 강조 (배경색 변경)

### 3.3 하단 네비게이션 (모바일 < 1024px)

```
+----------------------------------------------+
|  [할일]    [카테고리]    [프로필]              |
+----------------------------------------------+
```

- 화면 하단 고정 (fixed bottom)
- 아이콘 + 텍스트 레이블 조합

### 3.4 모달 오버레이 기본 구조

```
+------------------------------------------------------------+
|  ████████████████████████ (반투명 오버레이)                |
|  +--------------------------------------------------------+ |
|  |  모달 제목                                     [ X ]  | |
|  +--------------------------------------------------------+ |
|  |                                                        | |
|  |  모달 콘텐츠 영역                                      | |
|  |                                                        | |
|  +--------------------------------------------------------+ |
|  |                  [ 취소 ]        [ 확인 ]              | |
|  +--------------------------------------------------------+ |
+------------------------------------------------------------+
```

- 오버레이 클릭 시 모달 닫기 (취소와 동일)
- ESC 키로 닫기 지원
- X 버튼은 우측 상단 고정
- 모달 박스 내부 너비: 60자 기준 통일

### 3.5 토스트 알림

```
+-----------------------------------------------+
|  v  할일이 등록되었습니다.          [ X ]     |
+-----------------------------------------------+
```

- 화면 우측 하단 또는 상단에 3초간 표시 후 자동 소멸
- 성공(녹색), 오류(적색), 안내(파란색) 세 가지 유형

### 3.6 에러 메시지 인라인 표시

```
  [___________________]
  ! 이메일 형식이 올바르지 않습니다.
```

- 입력 필드 하단에 붉은 텍스트로 표시
- 필드 테두리도 함께 적색으로 변경

### 3.7 로딩 인디케이터

#### 스피너

```
+------------------------------------------------------------+
|                                                            |
|                      ⟳  불러오는 중...                    |
|                                                            |
+------------------------------------------------------------+
```

- 중앙 정렬, 페이지 또는 섹션 단위 전체 로딩 시 사용

#### 스켈레톤 카드

```
+-- 할일 카드 (로딩 중) ----------------------------------------+
|  ░░  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░      [    ]    |
|       ░░░░░░░░░░░░  마감: ░░░░░░░░░░                        |
|       ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░              [    ]    |
+--------------------------------------------------------------+
```

- 목록 로딩 시 카드 3개를 스켈레톤으로 먼저 표시
- 실제 데이터 수신 완료 후 실제 카드로 교체

#### 하단 무한 스크롤 로딩

```
|                  ⟳  더 불러오는 중...                       |
```

- 무한 스크롤 방식에서 추가 데이터 요청 중일 때 목록 하단에 표시

### 4.1 SCR-01 회원가입 화면

**관련 유스케이스:** US-U-01
**관련 시나리오:** SCN-U-01, SCN-INTEGRATED-01

#### 데스크톱 와이어프레임

```
+------------------------------------------------------------------------------+
|                              TodoListApp                                     |
+------------------------------------------------------------------------------+
|                                                                              |
|                  +--------------------------------------+                   |
|                  |            회원가입                  |                   |
|                  +--------------------------------------+                   |
|                  |                                      |                   |
|                  |  이메일                              |                   |
|                  |  [__________________________________]|                   |
|                  |                                      |                   |
|                  |  비밀번호  ※ 8자 이상               |                   |
|                  |  [__________________________________]|                   |
|                  |                                      |                   |
|                  |  비밀번호 확인                       |                   |
|                  |  [__________________________________]|                   |
|                  |                                      |                   |
|                  |          [ 회원가입 ]                |                   |
|                  |                                      |                   |
|                  |  이미 계정이 있으신가요?             |                   |
|                  |       [ 로그인으로 이동 ]            |                   |
|                  |                                      |                   |
|                  +--------------------------------------+                   |
|                                                                              |
+------------------------------------------------------------------------------+
```

#### 에러 상태 와이어프레임

```
|                  +--------------------------------------+                   |
|                  |            회원가입                  |                   |
|                  +--------------------------------------+                   |
|                  |                                      |                   |
|                  |  이메일                              |                   |
|                  |  [user@example.com_______________]   |  <- 테두리 정상   |
|                  |                                      |                   |
|                  |  비밀번호  ※ 8자 이상               |                   |
|                  |  [*******________________________]   |  <- 테두리 적색   |
|                  |  ! 비밀번호는 최소 8자 이상이어야 합니다.               |
|                  |                                      |                   |
|                  |  비밀번호 확인                       |                   |
|                  |  [*****__________________________]   |  <- 테두리 적색   |
|                  |  ! 비밀번호가 일치하지 않습니다.                         |
|                  |                                      |                   |
|                  |          [ 회원가입 ]                |                   |
|                  +--------------------------------------+                   |
```

#### 화면 요소 설명

| 요소               | 유형       | 필수 | 제약 조건                                 |
|--------------------|------------|------|-------------------------------------------|
| 이메일             | text input | 필수 | RFC 5322 이메일 형식 검증 (BR-U-01)       |
| 비밀번호           | password input | 필수 | 최소 8자 이상 (BR-U-02)               |
| 비밀번호 확인      | password input | 필수 | 비밀번호 필드와 동일해야 함              |
| 회원가입 버튼      | button     | -    | 모든 필드 유효 시 활성화                  |
| 로그인으로 이동    | link       | -    | SCR-02 로그인 화면으로 이동               |

#### 인터랙션 노트

- 이메일 중복 시: "이미 사용 중인 이메일입니다" (서버 응답)
- 비밀번호 8자 미만: "비밀번호는 최소 8자 이상이어야 합니다"
- 비밀번호 불일치: "비밀번호가 일치하지 않습니다"
- 성공 시: 토스트 표시 후 SCR-02 로그인 화면으로 자동 이동

#### 화면 전이

| 트리거              | 목적지       |
|---------------------|--------------|
| 회원가입 성공       | SCR-02 로그인 |
| "로그인으로 이동"   | SCR-02 로그인 |

### 4.2 SCR-02 로그인 화면

**관련 유스케이스:** US-U-02
**관련 시나리오:** SCN-U-02, SCN-INTEGRATED-01

#### 데스크톱 와이어프레임

```
+------------------------------------------------------------------------------+
|                              TodoListApp                                     |
+------------------------------------------------------------------------------+
|                                                                              |
|                  +--------------------------------------+                   |
|                  |              로그인                  |                   |
|                  +--------------------------------------+                   |
|                  |                                      |                   |
|                  |  이메일                              |                   |
|                  |  [__________________________________]|                   |
|                  |                                      |                   |
|                  |  비밀번호                            |                   |
|                  |  [__________________________________]|                   |
|                  |                                      |                   |
|                  |          [ 로그인 ]                  |                   |
|                  |                                      |                   |
|                  |  계정이 없으신가요?                  |                   |
|                  |       [ 회원가입으로 이동 ]          |                   |
|                  |                                      |                   |
|                  +--------------------------------------+                   |
|                                                                              |
+------------------------------------------------------------------------------+
```

#### 세션 만료 리다이렉트 상태 (상단 토스트)

```
+------------------------------------------------------------------------------+
|  ! 세션이 만료되었습니다. 다시 로그인해주세요.                    [ X ]     |
+------------------------------------------------------------------------------+
|                              TodoListApp                                     |
+------------------------------------------------------------------------------+
|                                                                              |
|                  +--------------------------------------+                   |
|                  |              로그인                  |                   |
|                  +--------------------------------------+                   |
|                  |                                      |                   |
|                  |  이메일                              |                   |
|                  |  [__________________________________]|                   |
|                  |                                      |                   |
|                  |  비밀번호                            |                   |
|                  |  [__________________________________]|                   |
|                  |                                      |                   |
|                  |          [ 로그인 ]                  |                   |
|                  +--------------------------------------+                   |
|                                                                              |
+------------------------------------------------------------------------------+
```

- 인증 만료(401) 후 SCR-02로 리다이렉트된 경우 화면 최상단에 경고 토스트 표시
- 토스트는 적색(오류) 유형, 사용자가 닫기 전까지 유지

#### 에러 상태 와이어프레임

```
|                  +--------------------------------------+                   |
|                  |              로그인                  |                   |
|                  +--------------------------------------+                   |
|                  |                                      |                   |
|                  |  이메일                              |                   |
|                  |  [user@example.com_______________]   |                   |
|                  |                                      |                   |
|                  |  비밀번호                            |                   |
|                  |  [*******________________________]   |                   |
|                  |                                      |                   |
|                  |  ! 이메일 또는 비밀번호가 올바르지 않습니다.             |
|                  |  ※ 보안상 이메일 존재 여부를 별도 안내하지 않음         |
|                  |                                      |                   |
|                  |          [ 로그인 ]                  |                   |
|                  +--------------------------------------+                   |
```

#### 화면 요소 설명

| 요소               | 유형           | 필수 | 제약 조건                              |
|--------------------|----------------|------|----------------------------------------|
| 이메일             | text input     | 필수 | 이메일 형식 클라이언트 검증            |
| 비밀번호           | password input | 필수 | 입력값 마스킹                          |
| 로그인 버튼        | button         | -    | 클릭 시 POST /api/v1/auth/login 호출   |
| 회원가입으로 이동  | link           | -    | SCR-01 이동                            |

#### 인터랙션 노트

- 인증 실패 시 단일 오류 메시지: "이메일 또는 비밀번호가 올바르지 않습니다"
  - 이메일 미존재와 비밀번호 불일치를 구분하여 응답하지 않음 (사용자 열거 방지, BR-U-01)
- 성공 시 JWT를 Zustand authStore 메모리에 저장 후 SCR-03으로 이동
- 이미 로그인 상태에서 접근 시 SCR-03으로 리다이렉트
- 세션 만료(401) 리다이렉트 시 상단 토스트로 만료 사유 안내

#### 화면 전이

| 트리거             | 목적지                |
|--------------------|-----------------------|
| 로그인 성공        | SCR-03 할일 목록      |
| "회원가입으로 이동" | SCR-01 회원가입       |
| 세션 만료 리다이렉트 | SCR-02 (상단 토스트) |

### 4.3 SCR-03 할일 목록 화면 (메인 대시보드)

**관련 유스케이스:** US-T-02, US-T-04, US-T-05, US-T-07
**관련 시나리오:** SCN-T-02, SCN-T-04, SCN-T-05, SCN-T-07, SCN-INTEGRATED-01

#### 데스크톱 와이어프레임

```
+------------------------------------------------------------------------------+
|  TodoListApp                                        [홍길동]  [ 로그아웃 ] |

|                    |                                                         |
|  내비게이션        |  할일 목록                         [ + 할일 추가 ]     |
|                    |  ------------------------------------------------------|
|  > 할일 목록       |                                                         |
|    카테고리 관리   |  +-- 필터 패널 ----------------------------------------+|
|    프로필 설정     |  | 카테고리: [ 전체 ▼ ]                               ||
|                    |  | 완료 여부: (●) 전체  ( ) 미완료  ( ) 완료          ||
|                    |  | 기간 (종료예정일):                                  ||
|                    |  |   시작일 [__________]  종료일 [__________]          ||
|                    |  |   ※ 종료예정일 없는 할일은 기간 필터 제외          ||
|                    |  |                        [ 필터 적용 ] [ 초기화 ]     ||
|                    |  +-----------------------------------------------------+|
|                    |                                                         |
|                    |  +-- 할일 카드 ----------------------------------------+|
```

- 탈퇴 성공 시: Zustand authStore 초기화 + "계정이 탈퇴되었습니다" 토스트 + SCR-02 이동

#### 화면 전이

| 트리거          | 목적지                            |
|-----------------|-----------------------------------|
| 탈퇴 완료 성공  | SCR-02 로그인 화면 (토큰 초기화)  |
| 취소 / X        | 모달 닫기 -> SCR-10 프로필 설정   |

## 7. 접근성 및 사용성 기준

### 7.1 키보드 포커스 순서 (Tab 순서)

- 포커스 순서는 시각적 레이아웃 흐름(좌상단 -> 우하단)을 따른다.
- 모달이 열리면 포커스는 모달 첫 번째 인터랙티브 요소로 이동하고, 닫히면 모달을 트리거한 요소로 복귀한다.
- 모달 내부에서 Tab 이동은 모달 내부 요소로 제한(포커스 트랩)한다.
- 사이드바 네비게이션 -> 헤더 -> 메인 콘텐츠 순서로 포커스를 구성한다.
- `tabindex="-1"` 을 사용하여 비활성화 요소는 Tab 순서에서 제외한다.

### 7.2 aria-live 영역

| 영역            | role / aria 속성         | 설명                                       |
|-----------------|--------------------------|--------------------------------------------|
| 에러 메시지     | `role="alert"`           | 인라인 에러 발생 시 즉시 스크린 리더 읽음 |
| 토스트 알림     | `aria-live="assertive"`  | 성공/오류 토스트 발생 시 즉시 읽음        |
| 로딩 상태       | `aria-live="polite"`     | 스켈레톤/스피너 상태 변경 시 읽음         |
| 목록 갱신       | `aria-live="polite"`     | 할일/카테고리 목록 변경 시 읽음           |

### 7.3 모바일 터치 영역

- 모든 인터랙티브 요소(버튼, 체크박스, 링크)의 터치 영역은 최소 44×44px을 확보한다.
- 체크박스 터치 영역은 체크박스 아이콘 자체가 아닌 카드 좌측 영역 전체로 확장한다.
- 삭제/편집 버튼은 터치 영역 44px 확보를 위해 패딩을 충분히 부여한다.

### 7.4 색상 대비 기준 (WCAG AA)

| 요소                  | 최소 대비율 | 기준                   |
|-----------------------|-------------|------------------------|
| 일반 텍스트           | 4.5:1       | WCAG 2.1 AA            |
| 대형 텍스트 (18px+)   | 3:1         | WCAG 2.1 AA            |
| UI 컴포넌트 경계      | 3:1         | WCAG 2.1 AA            |
| 비활성화(disabled) 요소 | 예외 적용 | 비활성 상태 시각 구분  |

- 카테고리 색상 뱃지: 배경색 위 텍스트가 4.5:1 이상이어야 하며, 밝은 색상 배경에는 어두운 텍스트를 사용한다.
- 에러 메시지 텍스트(적색): 배경 대비 4.5:1 이상 확보.

### 7.5 라벨-입력 필드 연결

- 모든 입력 필드는 `<label for="...">` / `htmlFor`를 통해 레이블과 명시적으로 연결한다.
- 플레이스홀더(placeholder)는 레이블을 대체하지 않으며, 보조 안내 텍스트로만 사용한다.
- 아이콘만 있는 버튼(X 닫기, 편집, 삭제)에는 `aria-label` 속성으로 의미를 제공한다.
  - 예: `<button aria-label="모달 닫기">X</button>`
- 체크박스에는 할일 제목을 `aria-label`로 연결하여 "Q2 프로젝트 기획서 작성 완료 여부" 형태로 읽히도록 한다.
