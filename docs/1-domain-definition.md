# TodoListApp 도메인 정의서

> 버전: 1.0.0
> 작성일: 2026-05-12
> 상태: 초안 (Draft)

---

## 1. 프로젝트 개요

### 목적

개인별로 할일을 안전하게 관리할 수 있는 인증 기반 할일 관리 애플리케이션을 제공한다. 사용자는 회원가입 및 로그인을 통해 자신만의 데이터 공간을 보유하며, 카테고리를 활용한 체계적인 할일 분류와 다양한 필터링 기능으로 생산성을 향상시킨다.

### 범위

| 구분 | 포함 | 제외 |
|------|------|------|
| 사용자 관리 | 회원가입, 로그인, 개인정보 수정 | 소셜 로그인, 권한 관리 |
| 할일 관리 | 등록, 수정, 삭제, 조회, 완료 처리 | 공유, 협업, 댓글 |
| 카테고리 관리 | 기본 카테고리 제공, 사용자 정의 카테고리 추가/수정/삭제 | 카테고리 공유 |
| 필터링 | 카테고리별, 기간별, 완료 여부별 | 태그, 우선순위 기반 정렬 |

## 2. 도메인 용어 정의 (Ubiquitous Language)

| 용어 | 영문 | 정의 |
|------|------|------|
| 사용자 | User | 회원가입을 완료하고 인증을 통해 앱을 사용하는 주체 |
| 인증 | Authentication | 사용자의 신원을 이메일과 비밀번호로 확인하는 절차 |
| 할일 | Todo | 사용자가 수행해야 할 단위 작업 항목 |
| 할일 제목 | Todo Title | 할일을 식별하는 짧고 명확한 이름 |
| 설명 | Description | 할일의 세부 내용을 기술하는 선택적 텍스트 |
| 종료예정일 | Due Date | 할일을 완료해야 하는 목표 날짜 |
| 완료 여부 | Completion Status | 할일이 완료되었는지 나타내는 상태 (완료 / 미완료) |
| 카테고리 | Category | 할일을 분류하는 그룹 단위 |
| 기본 카테고리 | Default Category | 시스템이 제공하는 기본 분류 항목 (예: 개인, 업무, 쇼핑) |
| 사용자 정의 카테고리 | Custom Category | 사용자가 직접 생성한 카테고리 |
| 기간 필터 | Period Filter | 종료예정일 범위를 기준으로 할일 목록을 좁히는 조건 |

## 3. 핵심 도메인 모델

### 엔티티 (Entity)

#### User (사용자)
- **식별자**: userId (UUID)
- **속성**: 이메일, 비밀번호(해시), 이름, 가입일시, 최종수정일시
- **역할**: 모든 도메인 데이터의 소유자

#### Todo (할일)
- **식별자**: todoId (UUID)
- **속성**: 제목, 설명, 종료예정일, 완료 여부, 완료일시, 생성일시, 최종수정일시
- **연관**: userId (소유자), categoryId (분류)

#### Category (카테고리)
- **식별자**: categoryId (UUID)
- **속성**: 이름, 색상코드, 기본 카테고리 여부, 생성일시
- **연관**: userId (소유자, 사용자 정의 카테고리의 경우)

### 값 객체 (Value Object)

| 값 객체 | 구성 요소 | 설명 |
|---------|-----------|------|
| Email | 이메일 문자열 | 형식 유효성 검증 포함, 불변 |
| DueDate | 날짜 | 과거 날짜 허용, 비교 연산 지원 |
| CompletionStatus | 완료 여부 (Boolean), 완료일시 | 미완료↔완료 양방향 전이 가능 (BR-T-03 참조) |

### 집계 루트 (Aggregate Root)

```
User (집계 루트)
 └── 소유: Todo (여러 개)
 └── 소유: Category (사용자 정의, 여러 개)

Category (독립 집계)
 └── 기본 카테고리: 시스템 관리
 └── 사용자 정의 카테고리: 해당 User가 관리
```

> 핵심 원칙: Todo는 반드시 User에 속하며, 다른 사용자의 Todo에 접근할 수 없다.

## 4. 비즈니스 규칙

### 사용자 규칙

- BR-U-01: 이메일은 시스템 전체에서 고유해야 한다.
- BR-U-02: 비밀번호는 최소 8자 이상이어야 하며, 평문으로 저장하지 않는다.
- BR-U-03: 인증되지 않은 사용자는 할일 및 카테고리 데이터에 접근할 수 없다.

### 할일 규칙

- BR-T-01: 할일 제목은 필수값이며, 1자 이상 200자 이하이어야 한다.
- BR-T-02: 할일은 반드시 하나의 카테고리에 속해야 한다.
- BR-T-03: 완료 처리된 할일은 완료 취소(미완료 복원)가 가능하다.
- BR-T-04: 사용자는 자신이 소유한 할일만 조회, 수정, 삭제할 수 있다.
- BR-T-05: 종료예정일은 선택 항목이며, 설정하지 않을 수 있다.

### 카테고리 규칙

- BR-C-01: 기본 카테고리는 수정 및 삭제할 수 없다.
- BR-C-02: 사용자 정의 카테고리 이름은 해당 사용자 내에서 중복될 수 없다.
- BR-C-03: 할일이 연결된 카테고리는 삭제할 수 없다. (연결된 할일 재분류 후 삭제 가능)
- BR-C-04: 사용자 정의 카테고리는 해당 사용자만 수정 및 삭제할 수 있다.

### 필터링 규칙

- BR-F-01: 필터는 단독 또는 복합 조건으로 적용 가능하다.
- BR-F-02: 기간 필터는 종료예정일을 기준으로 적용하며, 종료예정일이 없는 할일은 기간 필터 결과에서 제외된다.
- BR-F-03: 필터 미적용 시 전체 할일 목록을 생성일시 역순으로 반환한다.

## 5. 주요 유스케이스 (사용자 스토리)

### 사용자 관리

| ID | 사용자 스토리 | 인수 조건 |
|----|--------------|-----------|
| US-U-01 | 신규 사용자로서, 이메일과 비밀번호로 회원가입하여 서비스를 이용하고 싶다. | 이메일 중복 검증 후 계정 생성, 가입 완료 응답 반환 |
| US-U-02 | 가입된 사용자로서, 이메일과 비밀번호로 로그인하여 인증 토큰을 발급받고 싶다. | 자격 증명 일치 시 토큰 발급, 불일치 시 오류 반환 |
| US-U-03 | 인증된 사용자로서, 내 이름 또는 비밀번호를 변경하고 싶다. | 변경 항목만 업데이트, 이메일 변경 불가 |
| US-U-04 | 인증된 사용자로서, 서비스를 더 이상 이용하지 않아 계정을 탈퇴하고 싶다. | 본인 확인 후 계정 및 연관 데이터(할일·카테고리) 즉시 영구 삭제 |

### 카테고리 관리

| ID | 사용자 스토리 | 인수 조건 |
|----|--------------|-----------|
| US-C-01 | 인증된 사용자로서, 나만의 카테고리를 추가하여 할일을 체계적으로 분류하고 싶다. | 이름 중복 검증 후 카테고리 생성, 기본 카테고리와 함께 조회 가능 |
| US-C-02 | 인증된 사용자로서, 내가 만든 카테고리의 이름이나 색상을 수정하고 싶다. | 기본 카테고리는 수정 불가, 사용자 정의 카테고리만 수정 허용 |
| US-C-03 | 인증된 사용자로서, 더 이상 사용하지 않는 카테고리를 삭제하고 싶다. | 연결된 할일 없을 경우에만 삭제 허용 |

### 할일 관리

| ID | 사용자 스토리 | 인수 조건 |
|----|--------------|-----------|
| US-T-01 | 인증된 사용자로서, 제목, 설명, 종료예정일, 카테고리를 지정하여 할일을 등록하고 싶다. | 제목, 카테고리 필수 검증 후 할일 생성 |
| US-T-02 | 인증된 사용자로서, 내 할일 목록을 조회하고 싶다. | 본인 소유 할일만 반환, 생성일시 역순 기본 정렬 |
| US-T-03 | 인증된 사용자로서, 할일의 내용을 수정하고 싶다. | 본인 소유 검증 후 수정 허용 |
| US-T-04 | 인증된 사용자로서, 완료한 할일을 완료 처리하고 싶다. | 완료 여부 및 완료일시 업데이트 |
| US-T-05 | 인증된 사용자로서, 잘못 완료 처리한 할일을 미완료로 되돌리고 싶다. | 완료 여부 해제 및 완료일시 초기화 |
| US-T-06 | 인증된 사용자로서, 필요 없는 할일을 삭제하고 싶다. | 본인 소유 검증 후 영구 삭제 |
| US-T-07 | 인증된 사용자로서, 카테고리, 기간, 완료 여부로 할일을 필터링하여 조회하고 싶다. | 복합 필터 조건 지원, 조건에 해당하는 할일만 반환 |

## 6. 도메인 이벤트

| 이벤트명 | 발생 시점 | 주요 데이터 |
|----------|-----------|-------------|
| UserRegistered | 회원가입 완료 시 | userId, email, registeredAt |
| UserLoggedIn | 로그인 성공 시 | userId, loginAt |
| UserProfileUpdated | 개인정보 수정 시 | userId, updatedFields, updatedAt |
| CategoryCreated | 사용자 정의 카테고리 생성 시 | categoryId, userId, name, createdAt |
| CategoryUpdated | 카테고리 수정 시 | categoryId, userId, updatedFields, updatedAt |
| CategoryDeleted | 카테고리 삭제 시 | categoryId, userId, deletedAt |
| TodoCreated | 할일 등록 시 | todoId, userId, categoryId, title, dueDate, createdAt |
| TodoUpdated | 할일 수정 시 | todoId, userId, updatedFields, updatedAt |
| TodoCompleted | 할일 완료 처리 시 | todoId, userId, completedAt |
| TodoReopened | 할일 완료 취소 시 | todoId, userId, reopenedAt |
| TodoDeleted | 할일 삭제 시 | todoId, userId, deletedAt |
