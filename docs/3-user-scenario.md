## 복합 시나리오

### SCN-INTEGRATED-01: 신입 사원의 첫 업무 설정

**시나리오 설명:** 새로 입사한 30대 직장인이 시스템에 가입하고, 로그인 후 회사 업무를 관리하기 위해 카테고리를 생성한 후, 첫 업무 과제들을 등록하고 완료 처리하는 전체 사용 흐름

**행위자:** 신입 사원 김철수(33세)

**시나리오 흐름:**

1. 김철수는 투두리스트 앱에 접속하여 회원가입 버튼을 클릭한다.
2. 이메일(chulsu.kim@company.com)과 비밀번호(MyPassword123)를 입력하고 가입 요청을 한다.
3. 시스템에서 이메일 중복 여부를 확인하고 계정을 생성한다.
4. 회원가입이 완료되었다는 확인 메시지를 받는다.
5. 로그인 화면으로 이동하여 동일한 이메일과 비밀번호로 로그인한다.
6. 로그인이 성공하고 메인 대시보드로 진입한다.
7. 기본 카테고리(개인, 업무, 쇼핑)가 자동으로 생성되어 있는 것을 확인한다.
8. "프로젝트 관리" 카테고리를 새로 생성한다.
9. "OKR 달성 추적" 카테고리도 추가로 생성한다.
10. 할일 추가 버튼을 클릭하여 첫 번째 과제를 등록한다.
    - 제목: "Q2 프로젝트 기획서 작성"
    - 카테고리: "프로젝트 관리"
    - 설명: "신규 프로젝트 기획서 초안 작성"
    - 종료예정일: 2026-05-30
11. 두 번째 과제를 등록한다.
    - 제목: "팀 미팅 참석"
    - 카테고리: "업무"
    - 종료예정일: 2026-05-15
12. 세 번째 과제를 등록한다.
    - 제목: "신입사원 교육 이수"
    - 카테고리: "개인"
13. 할일 목록을 조회하면 최신 순으로 세 개의 과제가 표시된다.
14. "팀 미팅 참석" 과제를 완료 처리한다.
15. 완료된 과제의 상태가 "완료"로 표시되고 완료일시가 기록된다.
16. 나중에 실수로 "팀 미팅 참석"을 완료 취소한다.
17. 과제 상태가 다시 "미완료"로 돌아간다.
18. 할일 목록을 카테고리별로 필터링하여 "업무" 카테고리 할일만 조회한다.

**사후 조건:**
- 김철수의 계정이 정상적으로 생성되고 인증 상태 유지
- 2개의 사용자 정의 카테고리 생성 완료 (기본 카테고리 3개 포함 총 5개 보유)
- 3개의 할일 등록 완료
- 완료 처리 및 취소 기능 정상 작동
- 필터링 기능으로 특정 카테고리의 할일만 조회 가능

## 사용자 관리 시나리오

### SCN-U-01: 회원가입 - 정상 흐름

**시나리오 ID:** SCN-U-01  
**관련 유스케이스:** US-U-01  

**사전 조건:**
- 사용자가 앱에 최초 접근하여 회원가입 화면을 열고 있는 상태

**행위자:** 신규 사용자

**정상 흐름:**

1. 사용자가 회원가입 버튼을 클릭한다.
2. 이메일 입력 필드에 이메일 주소를 입력한다.
3. 비밀번호 입력 필드에 8자 이상의 비밀번호를 입력한다.
4. 비밀번호 확인 필드에 동일한 비밀번호를 재입력한다.
5. 회원가입 버튼을 클릭한다.
6. 시스템에서 입력된 이메일이 기존 계정에 등록되지 않았는지 확인한다.
7. 비밀번호가 최소 8자 요구사항을 충족하는지 검증한다.
8. 계정 생성이 완료된다.
9. 확인 메시지가 표시되고 로그인 화면으로 이동한다.

**예외 흐름:**

- **만약 입력된 이메일이 이미 등록된 경우:**
  1. 시스템에서 "이미 사용 중인 이메일입니다"라는 오류 메시지를 표시한다.
  2. 사용자는 다른 이메일로 다시 입력할 수 있다.

- **만약 비밀번호가 8자 미만인 경우:**
  1. 시스템에서 "비밀번호는 최소 8자 이상이어야 합니다"라는 검증 오류를 표시한다.
  2. 사용자는 비밀번호를 다시 입력할 수 있다.

- **만약 두 비밀번호 입력이 일치하지 않는 경우:**
  1. 시스템에서 "비밀번호가 일치하지 않습니다"라는 오류 메시지를 표시한다.
  2. 사용자는 비밀번호를 재확인하여 다시 입력한다.

**사후 조건:**
- 신규 사용자 계정이 데이터베이스에 생성됨
- 이메일과 비밀번호(암호화된 형태)가 저장됨
- 사용자가 로그인 화면으로 안내됨

### SCN-U-02: 로그인 - 정상 흐름

**시나리오 ID:** SCN-U-02  
**관련 유스케이스:** US-U-02  

**사전 조건:**
- 이미 등록된 사용자 계정이 존재
- 사용자가 로그인 화면에 접속한 상태

**행위자:** 기존 사용자

**정상 흐름:**

1. 사용자가 이메일 입력 필드에 등록된 이메일을 입력한다.
2. 비밀번호 입력 필드에 정확한 비밀번호를 입력한다.
3. 로그인 버튼을 클릭한다.
4. 시스템에서 입력된 이메일이 등록되어 있는지 확인한다.
5. 입력된 비밀번호와 저장된 비밀번호를 비교 검증한다.
6. 자격 증명이 일치하여 로그인이 성공한다.
7. 시스템에서 사용자를 식별하는 인증 토큰을 발급한다.
8. 사용자가 메인 대시보드로 이동한다.

**예외 흐름:**

- **만약 이메일 또는 비밀번호가 올바르지 않은 경우:**
  1. 시스템에서 "이메일 또는 비밀번호가 올바르지 않습니다"라는 단일 오류 메시지를 표시한다.
  2. 사용자는 이메일과 비밀번호를 다시 입력할 수 있다.
  > 이메일 미존재와 비밀번호 불일치를 구분하여 응답하지 않는다. 구분 시 계정 존재 여부가 노출되는 보안 취약점(사용자 열거)이 발생한다.

**사후 조건:**
- 사용자가 인증 토큰을 받아 로그인 상태 유지
- 메인 대시보드에 접근 가능한 상태
- 본인 소유 데이터에 한해 접근 가능한 상태

### SCN-U-03: 프로필 수정 - 정상 흐름

**시나리오 ID:** SCN-U-03  
**관련 유스케이스:** US-U-03  

**사전 조건:**
- 사용자가 로그인한 상태
- 프로필 설정 페이지에 접속한 상태

**행위자:** 로그인한 사용자

**정상 흐름:**

1. 사용자가 프로필 설정 페이지에 접속한다.
2. 현재 등록된 이름과 이메일을 확인한다.
3. 이름 수정 버튼을 클릭한다.
4. 새로운 이름(예: "김철수")을 입력한다.
5. 저장 버튼을 클릭한다.
6. 시스템에서 변경사항을 업데이트한다.
7. "프로필이 수정되었습니다"라는 확인 메시지가 표시된다.

**대안 흐름 (비밀번호 변경):**

1. 사용자가 비밀번호 변경 버튼을 클릭한다.
2. 현재 비밀번호 입력 필드에 기존 비밀번호를 입력한다.
3. 새로운 비밀번호(8자 이상)를 입력한다.
4. 새로운 비밀번호를 다시 입력하여 확인한다.
5. 저장 버튼을 클릭하면 비밀번호가 변경된다.

**예외 흐름:**

- **만약 새 비밀번호가 8자 미만인 경우:**
  1. 시스템에서 "비밀번호는 최소 8자 이상이어야 합니다"라는 검증 오류를 표시한다.
  2. 사용자는 비밀번호를 다시 입력할 수 있다.

- **만약 이메일 변경을 시도하는 경우:**
  1. 이메일 입력 필드가 비활성화되어 있음을 확인한다.
  2. "이메일은 변경할 수 없습니다"라는 안내 메시지를 확인한다.

**사후 조건:**
- 사용자의 프로필 정보(이름 또는 비밀번호)가 업데이트됨
- 변경사항이 데이터베이스에 저장됨
- 이메일은 변경되지 않은 상태 유지

### SCN-U-04: 회원 탈퇴 - 정상 흐름

**시나리오 ID:** SCN-U-04  
**관련 유스케이스:** US-U-04  

**사전 조건:**
- 사용자가 로그인한 상태
- 설정 페이지에 접속한 상태

**행위자:** 로그인한 사용자

**정상 흐름:**

1. 사용자가 설정 페이지에서 회원 탈퇴 옵션을 찾는다.
2. 회원 탈퇴 버튼을 클릭한다.
3. "회원 탈퇴 후에는 모든 데이터가 삭제되며 복구할 수 없습니다"라는 경고 메시지가 표시된다.
4. 계속 진행하시겠습니까?라는 확인 대화상자가 나타난다.
5. 사용자가 "확인" 버튼을 클릭한다.
6. 본인 확인을 위해 현재 비밀번호를 입력하는 페이지로 이동한다.
7. 사용자가 정확한 비밀번호를 입력한다.
8. 최종 확인 버튼을 클릭한다.
9. 시스템에서 비밀번호를 검증한다.
10. 사용자 계정, 관련 모든 할일, 모든 카테고리가 즉시 영구 삭제된다.
11. "계정이 탈퇴되었습니다"라는 메시지와 함께 로그인 화면으로 리다이렉트된다.

**예외 흐름:**

- **만약 비밀번호 입력이 일치하지 않는 경우:**
  1. 시스템에서 "비밀번호가 일치하지 않습니다"라는 오류를 표시한다.
  2. 사용자는 비밀번호를 다시 입력할 기회가 제공된다.

**사후 조건:**
- 사용자 계정이 데이터베이스에서 완전히 삭제됨
- 해당 사용자의 모든 할일 데이터 삭제
- 해당 사용자의 모든 카테고리 데이터 삭제
- 로그인 상태가 해제됨

## 카테고리 관리 시나리오

### SCN-C-01: 카테고리 생성 - 정상 흐름

**시나리오 ID:** SCN-C-01  
**관련 유스케이스:** US-C-01  

**사전 조건:**
- 사용자가 로그인한 상태
- 카테고리 관리 페이지에 접속한 상태
- 기본 카테고리(개인, 업무, 쇼핑)가 이미 생성되어 있음

**행위자:** 로그인한 사용자

**정상 흐름:**

1. 사용자가 카테고리 목록을 확인한다.
2. 기본 카테고리 3개(개인, 업무, 쇼핑)가 표시된다.
3. 사용자가 "새 카테고리 추가" 버튼을 클릭한다.
4. 카테고리 이름 입력 필드에 "프로젝트 관리"를 입력한다.
5. 카테고리 색상 선택 옵션에서 파란색을 선택한다.
6. 저장 버튼을 클릭한다.
7. 시스템에서 해당 사용자의 기존 카테고리와 이름이 중복되지 않았는지 확인한다.
8. 새로운 카테고리가 생성되고 목록에 추가된다.
9. "카테고리가 생성되었습니다"라는 확인 메시지가 표시된다.

**예외 흐름:**

- **만약 이미 존재하는 카테고리 이름으로 생성하려는 경우:**
  1. 시스템에서 "이미 사용 중인 카테고리 이름입니다"라는 오류 메시지를 표시한다.
  2. 사용자는 다른 이름으로 다시 입력할 수 있다.

- **만약 카테고리 이름을 비워두고 저장하려는 경우:**
  1. 시스템에서 "카테고리 이름을 입력해주세요"라는 검증 오류를 표시한다.

**사후 조건:**
- 새로운 사용자 정의 카테고리가 데이터베이스에 생성됨
- 사용자의 카테고리 목록에 새 카테고리가 추가됨
- 새 카테고리를 할일 등록 시 선택할 수 있는 상태

### SCN-C-02: 카테고리 수정 - 정상 흐름

**시나리오 ID:** SCN-C-02  
**관련 유스케이스:** US-C-02  

**사전 조건:**
- 사용자가 로그인한 상태
- 카테고리 관리 페이지에 접속한 상태
- 최소 1개의 사용자 정의 카테고리가 존재

**행위자:** 로그인한 사용자

**정상 흐름:**

1. 사용자가 카테고리 목록에서 수정하려는 카테고리를 선택한다.
2. 카테고리 상세 페이지로 이동한다.
3. 수정 버튼을 클릭한다.
4. 카테고리 이름을 새로운 이름으로 변경한다(예: "프로젝트 관리" → "주요 프로젝트").
5. 카테고리 색상을 다른 색상으로 변경한다(예: 파란색 → 빨간색).
6. 저장 버튼을 클릭한다.
7. 시스템에서 변경사항을 검증 및 업데이트한다.
8. "카테고리가 수정되었습니다"라는 확인 메시지가 표시된다.

**예외 흐름:**

- **만약 기본 카테고리(개인, 업무, 쇼핑)를 수정하려는 경우:**

- **만약 이미 존재하는 카테고리 이름으로 수정하려는 경우:**
  1. 시스템에서 "이미 사용 중인 카테고리 이름입니다"라는 오류 메시지를 표시한다.
  2. 사용자는 다른 이름으로 수정할 수 있다.

**사후 조건:**
- 카테고리의 이름과 색상이 업데이트됨
- 변경사항이 데이터베이스에 저장됨

### SCN-C-03: 카테고리 삭제 - 정상 흐름

**시나리오 ID:** SCN-C-03  
**관련 유스케이스:** US-C-03  

**사전 조건:**
- 사용자가 로그인한 상태
- 카테고리 관리 페이지에 접속한 상태
- 최소 1개의 사용자 정의 카테고리가 존재
- 삭제 대상 카테고리에 연결된 할일이 없는 상태

**행위자:** 로그인한 사용자

**정상 흐름:**

1. 사용자가 카테고리 목록에서 삭제하려는 카테고리를 선택한다.
2. 카테고리 상세 페이지로 이동한다.
3. 삭제 버튼을 클릭한다.
4. "정말로 삭제하시겠습니까?"라는 확인 대화상자가 표시된다.
5. 사용자가 "삭제" 버튼을 클릭한다.
6. 시스템에서 해당 카테고리에 연결된 할일이 있는지 확인한다.
7. 연결된 할일이 없으므로 카테고리가 삭제된다.
8. "카테고리가 삭제되었습니다"라는 확인 메시지가 표시된다.
9. 사용자가 카테고리 목록으로 돌아간다.

**예외 흐름:**

- **만약 카테고리에 연결된 할일이 있는 경우:**
  1. 시스템에서 "이 카테고리에는 할일이 있어서 삭제할 수 없습니다"라는 오류 메시지를 표시한다.
  2. 사용자는 먼저 해당 카테고리의 모든 할일을 삭제하거나 다른 카테고리로 이동시켜야 한다는 안내를 받는다.

- **만약 기본 카테고리를 삭제하려는 경우:**
  1. 기본 카테고리의 삭제 버튼이 비활성화되어 있음을 확인한다.
  2. "기본 카테고리는 삭제할 수 없습니다"라는 안내 메시지가 표시된다.

**사후 조건:**
- 해당 카테고리가 데이터베이스에서 완전히 삭제됨
- 카테고리 목록에서 제거됨
- 향후 할일 등록 시 해당 카테고리를 선택할 수 없는 상태

**관련 유스케이스:** US-T-01

3. 제목 입력 필드에 "분기별 실적 보고서 작성"을 입력한다.

7. 저장 버튼을 클릭한다.
8. 시스템에서 제목 길이(1~200자)를 검증한다.

  2. 사용자는 제목을 입력할 수 있다.

- **만약 제목이 200자를 초과하는 경우:**
  1. 시스템에서 "제목은 최대 200자입니다"라는 오류 메시지를 표시한다.
  2. 사용자는 제목을 수정할 수 있다.

- **만약 카테고리를 선택하지 않은 경우:**
  1. 시스템에서 "카테고리를 선택해주세요"라는 검증 오류를 표시한다.
  2. 사용자는 카테고리를 선택해야 한다.

- **만약 설명과 종료예정일 없이 등록하려는 경우:**
  1. 설명과 종료예정일은 선택사항이므로 바로 등록된다.
  2. 시스템에서 제목과 카테고리 정보만으로 할일을 생성한다.

**사후 조건:**
- 새로운 할일이 데이터베이스에 생성됨
- 할일 목록에 최신 항목으로 표시됨
- 해당 할일에 생성일시가 자동으로 기록됨

  1. 시스템에서 "제목은 최대 200자입니다"라는 검증 오류를 표시한다.

### SCN-T-07: 할일 필터링 - 정상 흐름

**관련 유스케이스:** US-T-07

- 다양한 카테고리의 할일 5개 이상 존재

**정상 흐름:**

1. 사용자가 할일 목록 화면에 접속한다.
2. 화면 상단 또는 측면의 필터 옵션 영역을 확인한다.

*카테고리 필터 적용*

3. 카테고리 필터에서 "업무"를 선택한다.

12. 완료된 할일은 목록에서 제외된다.
